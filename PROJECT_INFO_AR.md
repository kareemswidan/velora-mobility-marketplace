# دليل مشروع Velora Mobility

**اسم المشروع:** Velora Mobility Marketplace  
**صاحب المشروع:** Kareem Swidan  
**نوع المشروع:** منصة Full-Stack عالمية متعددة الأطراف لخدمات الطاقة والعناية بالمركبات.

## فكرة المنصة

تخدم Velora فئتين رئيسيتين:

- العميل: يبحث عن أقرب محطة أو مغسلة أو مركز خدمة، يقارن الخدمات، يحجز موعدًا، يحفظ المواقع المفضلة، يدير مركباته، ويتسوق من E-Mart.
- صاحب العمل: يسجل منشأته وفروعها وخدماتها وأوقات العمل، ثم يدير الحجوزات بعد موافقة الإدارة.

توجد كذلك لوحة إدارة لمراجعة طلبات الشركاء، تحديث أسعار الوقود، إدارة الحجوزات والمنتجات، ومراجعة سجل العمليات.

## دورة الموافقة

1. يسجل صاحب العمل حسابًا ويرسل بيانات المنشأة.
2. تُحفظ المنشأة بحالة `PENDING` ولا تظهر للجمهور.
3. يراجع المدير الطلب من لوحة الإدارة ويختار الموافقة أو الرفض مع سبب.
4. عند الموافقة تصبح الفروع والخدمات قابلة للاكتشاف والحجز.
5. يصل إشعار لصاحب العمل، ويُحفظ القرار في سجل التدقيق.

## البرامج والتقنيات المستخدمة

- Next.js 14 App Router وTypeScript لبناء الواجهة والخادم.
- React وFramer Motion للحركة والتفاعل.
- Tailwind CSS للتصميم المتجاوب.
- MySQL أو MariaDB مع Prisma ORM لقاعدة البيانات.
- Node.js وnpm للتشغيل وإدارة الحزم.
- OpenStreetMap للخريطة والاتجاهات بدون مفتاح مدفوع.
- JWT داخل Cookies آمنة، وbcrypt لتشفير كلمات المرور.
- Visual Studio Code أو أي محرر كود، وPowerShell أو Terminal لتشغيل الأوامر.
- Chrome أو Edge للاختبار، وPrisma Studio لمراجعة البيانات بصريًا عند الحاجة.

## الصفحات الأساسية

- `/`: الصفحة الرئيسية والأسعار الحية ومقدمة الخدمات.
- `/stations`: البحث والتصفية وتحديد الأقرب والخريطة.
- `/stations/[id]`: تفاصيل الموقع والخدمات والمراجعات والمفضلة.
- `/booking`: إنشاء حجز مع التحقق من الوقت والتعارض.
- `/store`: متجر وسلة وطلب تجريبي يخصم المخزون.
- `/partners`: تسجيل أصحاب المحطات والمغاسل ومراكز الخدمة.
- `/account`: حجوزات العميل ومركباته وطلباته وإشعاراته ومفضلاته.
- `/business`: إدارة الفروع والخدمات والمواعيد والحجوزات.
- `/admin`: الموافقات والأسعار والمنتجات والحجوزات وسجل التدقيق.

## التشغيل السريع

1. ثبّت Node.js وMySQL أو XAMPP.
2. فك ضغط المشروع وافتح المجلد في Terminal.
3. نفذ `npm install`.
4. انسخ `.env.example` إلى `.env` وحدّث رابط قاعدة البيانات والسر.
5. نفذ `npm run db:push` ثم `npm run db:seed`.
6. نفذ `npm run dev` وافتح `http://localhost:3000`.

راجع `README.md` للتفاصيل الكاملة وحسابات العرض التجريبية وأوامر الاختبار.
